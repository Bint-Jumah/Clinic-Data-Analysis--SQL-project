# 🏥 Clinic Management Data Analysis with SQL

## 📌 Project Overview

This project demonstrates my SQL skills by analyzing a simulated clinic management database. The goal was to answer real-world healthcare business questions using SQL queries, helping uncover insights into patient records, appointments, billing, insurance coverage, medications, and doctor performance.

The project showcases my ability to design database tables, write analytical SQL queries, perform joins across multiple tables, and extract meaningful insights from relational data.


## 🗂️ Database Schema

The database consists of six related tables:

* **Patients** – Patient demographics and insurance information
* **Appointments** – Appointment details and visit reasons
* **Doctors** – Doctor information and specialties
* **Diagnoses** – Patient diagnoses linked to appointments
* **Medications** – Prescribed medications and renewal dates
* **Billing** – Billing amounts, insurance coverage, and patient payments


## 🔍 Business Questions Solved

Some of the analyses performed include:

* Identifying patients whose insurance covered less than 70% of their medical bills.
* Listing diabetic patients together with their latest medication renewal date.
* Determining which doctor has the lowest no-show rate.
* Exploring patient, appointment, medication, billing, and doctor records using SQL queries.
* Performing data retrieval, filtering, joins, aggregations, and ordering.
